# LINE便当店机器人 - 商业化完善清单

## 🎯 优先级1：关键功能（必须）

### 1. **订单数据持久化** ⭐⭐⭐⭐⭐
- 现状：生成收据但未保存订单
- 需求：保存每笔订单到数据库
- 建议方案：
  ```
  数据库选择：
  - SQLite（轻量版）
  - PostgreSQL（推荐，Fly.io免费支持）
  - MongoDB（灵活）
  
  记录内容：
  - 订单号
  - 商品列表
  - 总价
  - 客户LINE ID
  - 时间戳
  - 是否已支付
  ```

### 2. **错误处理和重试机制** ⭐⭐⭐⭐
- 现状：API调用失败时无提示
- 改进：
  - 自动重试（3次）
  - 超时保护（10秒）
  - 用户友好的错误消息

### 3. **API成本控制** ⭐⭐⭐⭐
- 现状：每条消息都调用OpenRouter
- 监控字段：
  - 每日API调用次数
  - 成本预算
  - 调用失败率

## 🎯 优先级2：商业运营（重要）

### 4. **商家后台管理系统**
- 订单查看面板
- 日营业统计
- 客户订单记录
- 菜单配置修改
- 营业时间设置

### 5. **消息审核和改进**
- 用户可能发送歧义订单（如"来点便当"）
- 需要：
  - 提示用户确认订单
  - 推荐菜品列表
  - 预设快捷菜单

### 6. **支付集成**
- 现状：生成小票但未关联支付
- 需求：
  - LINE Pay 集成
  - 支付验证
  - 已支付标记

### 7. **库存管理**
- 防止超卖
- 断货提醒
- 菜品下架

## 🎯 优先级3：用户体验（提升）


### 10. **营业状态**
- 显示营业时间
- 休息日提示
